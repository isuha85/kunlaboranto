## **대상**

프로젝트환경에서 wsl2 또는 PowerShell 이 막혀있고, Windows Terminal 만 가능한 경우, 기존 Linux 터미널에서 사용했던 스크립트를 사용할 수가 없었다. <br>
이를 해소하기 위해서, 윈도우용 DbToys4xx 스크립트를 작성한다.

* Windows Terminal 환경에서 동작
* TUNER 용도
  * 테이블정보: `인덱스, 통계정보` 등이고, 직접 구현하기 보다는 가급적 DB엔진의 기본 기능을 이용한다.
  * 가독성 높은 PLAN 확보
  * Active Session / Lock Wait 등 모니터링
  * DB properties 비교


---

## 1. 사전작업

* 설치
  * 윈도용 GIT 설치<br>
  `MinGW` 가 함께 설치되며, `cat, grep, sed, find, awk 등` 을 사용할 수 있다.
  * 윈도용 PostgreSQL CLI (psql) 설치<br>
  `C:\Program Files\PostgreSQL\17\bin`

* [MinTool4PG-lastest.zip](..) 파일을 %USERPROFILE%\Desktop\DBA 경로에 압축 해제
* "사용자 환경 변수" 설정
  * DBAHOME : %USERPROFILE%\Desktop\DBA 경로
  * MINGW : <윈도용 GIT 설치경로>/usr/bin 경로<br>
  `C:\Program Files\Git\usr\bin`
  * PATH 환경변수에 경로 추가<br>
  a. `C:\Program Files\PostgreSQL\17\bin` <br>
  b. `%MINGW%` <br>
  c. `%DBAHOME%\bin`


## 2. `pgpass.conf` 파일 추가

* **설정**
  * 윈도우 시스템 환경 변수에 `PGPASSFILE`을 추가
  * **기본경로:** `%APPDATA%\postgresql\pgpass.conf`
  * **권장경로:** `%DBAHOME%\bin\config\pgpass.conf`
* **내용:** 여러 서버 정보를 한 파일에 기록
  ```markdown
  127.0.0.1:5432:*:postgres:passwd
  192.168.1.50:5432:test_db:dev_user:dev_pass
  ```
  > **Tip:** 파일 탐색기 주소창에 `%APPDATA%`를 입력한 후, `postgresql` 폴더를 직접 생성(없을 경우)하고 그 안에 파일을 생성
