### 사족 

재요청시에 

```
쉼표(,)를 앞으로 빼고, ON 절은 아래로, WHERE 1=1을 넣어서 영문 주석으로 정렬해줘."
즉 아래처럼 해주오

Comma-First: SELECT 절의 쉼표를 컬럼명 앞에 배치
Vertical Alignment: 별칭(AS)과 주석의 시작 위치를 수직으로 정렬
ON Clause: JOIN 바로 아래 줄에 ON 조건을 배치
WHERE 1=1: WHERE 절 시작 시 항상 1=1을 추가
Side Comments: -- 스타일의 영문 주석을 AND 구문 우측에 정렬
```

### SQL Formatting Standard Summary

| 구분 | 규칙 (Rules) | 예시 (Examples) |
| :--- | :--- | :--- |
| **선행 쉼표** | `SELECT` 절의 쉼표는 컬럼명 앞에 위치 | `, column_name` |
| **수직 정렬** | 컬럼명, `AS` 별칭, 주석 위치를 수직으로 정렬 | `col AS alias -- Comment` |
| **ON 절 배치** | `JOIN` 문 바로 아래 줄에 `ON` 조건을 배치 | `JOIN table` <br> `   ON condition ` |
| **WHERE 1=1** | `WHERE` 절 바로 다음에 항상 `1=1`을 추가 | `WHERE 1=1` |
| **주석 형식** | `--` 스타일의 라인 주석을 사용 | `-- English comment` |
| **주석 위치** | `AND` 문법의 우측 끝에 맞춰 수직 정렬 | `AND cond -- Comment` |
| **언어 및 표기** | 주석은 영문(English)으로 작성 | `/* (X) */` 대신 `-- (O)` |

### 적용 예시 (Template)

```SQL
SELECT a.column_1                               AS alias_1
     , a.column_2                               AS alias_2
  FROM table_a a
  JOIN table_b b 
    ON b.id = a.id
 WHERE 1=1
   AND a.status = 'active'                      -- 1. Filter active records
   AND b.type = 'normal'                        -- 2. Filter normal types
 ORDER BY alias_1
;
```

이 스타일은 코드의 가독성을 높일 뿐만 아니라, 특정 조건을 주석 처리하거나 새로운 컬럼을 추가할 때 발생할 수 있는 구문 오류를 최소화하는 매우 효율적인 방식
